Question 1: Environment Setup

Your teammate Sarah is new to conda and asks for help. She wants to create a new environment called "data-analysis" with Python 3.11 and install pandas version 2.1.0. She also heard about conda-forge and wants to use it.

Part A: What commands should Sarah run? Write them in order.

Part B: How can Sarah verify that her environment was created successfully and that pandas is the correct version?

Part C: If Sarah forgets the name of her environment later, how can she list all her environments?

Solution:

Part A:

Part B:

# Check if environment activated (should see (data-analysis) in prompt) # List packages to verify pandas version conda list pandas # Alternative: check Python and import pandas python -c "import pandas as pd; print(pd.__version__)"

Part C:

# List all environments conda env list # OR conda info --envs

Question 3: Environment Sharing Challenge

Your team is collaborating on a machine learning project. Alex created an environment and shared this environment.yaml file:

name: ml-project channels: - conda-forge - defaults dependencies: - python=3.10 - pandas - scikit-learn=1.3.0 - matplotlib - jupyter - pip - pip: - seaborn==0.12.0 - requests

Part A: What potential issues do you see with this environment file for team collaboration?

Part B: Your teammate Jamie is on Windows while Alex uses macOS. When Jamie tries to create the environment, some packages fail to install. What might be causing this and how would you fix it?

Part C: Write the commands to create this environment, add a new package (numpy=1.24.0), and update the environment file properly.

Solution:

Part A - Issues identified:

  • Missing version numbers for pandas, matplotlib, jupyter (not reproducible)
  • Two channels listed (conda-forge and defaults) may cause conflicts
  • Mixing conda and pip packages can cause dependency resolution issues
  • No Python patch version specified (3.10 vs 3.10.8)

Part B: Cross-platform issues likely caused by:

  • Some builds might be OS-specific when versions aren't pinned
  • Different architectures (Intel vs ARM vs x86)
  • The file might have been exported with conda env export (full export) instead of --from-history

Fix: Use --from-history flag and specify versions manually

Part C - Commands:

# Create environment from file conda env create -f environment.yaml # Activate environment conda activate ml-project # Add numpy conda install numpy=1.24.0 # Update environment file (recommended approach) # Edit environment.yaml manually to add numpy=1.24.0 # OR export with --from-history and edit conda env export --from-history -f environment.yaml # Better approach: manually maintain environment.yaml # Then recreate environment to test: conda env remove -n ml-project conda env create -f environment.yaml

Improved environment.yaml:

name: ml-project channels: - conda-forge dependencies: - python=3.10 - pandas=2.0.3 - numpy=1.24.0 - scikit-learn=1.3.0 - matplotlib=3.7.1 - jupyter=1.0.0 - seaborn=0.12.0 - requests=2.31.0

Question 4: RegEx Data Cleaning

Your team received a messy dataset with user contact information. You need to use regular expressions to clean and validate the data. Here are some sample entries:

john.doe@email.com Jane_Smith@company.org Contact: (555) 123-4567 Phone: 555.987.6543 555-111-2222 email: bob@domain.co.uk invalid-email@ (604) 555-0123

Part A: Write a regex pattern that matches valid email addresses from the sample data.

Part B: Write a regex pattern that matches all the phone number formats shown.

Part C: The dataset also contains dates in multiple formats. Write regex patterns for these date formats:

  • 2024-01-15 (ISO format)
  • 01/15/2024 (US format)
  • 15-Jan-2024 (Day-Month-Year with abbreviated month)
Team Discussion: Start by identifying the patterns in each format. What parts are consistent? What varies? Test your patterns with the online regex tools mentioned in class.

Solution:

Part A - Email Pattern:

# Basic email pattern [A-Za-z0-9._%-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,} # More specific (matches the valid ones in sample) [A-Za-z0-9._%-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,4} # Explanation: # [A-Za-z0-9._%-]+ : username part (letters, numbers, dots, underscores, %, -) # @ : literal @ symbol # [A-Za-z0-9.-]+ : domain name (letters, numbers, dots, hyphens) # \. : literal dot (escaped) # [A-Za-z]{2,4} : top-level domain (2-4 letters)

Part B - Phone Number Pattern:

# Pattern for all phone formats shown (\([0-9]{3}\)\s?|[0-9]{3}[-.]?)[0-9]{3}[-.]?[0-9]{4} # Alternative - simpler approach matching each format separately: # Format 1: (555) 123-4567 \([0-9]{3}\)\s[0-9]{3}-[0-9]{4} # Format 2: 555.987.6543 [0-9]{3}\.[0-9]{3}\.[0-9]{4} # Format 3: 555-111-2222 [0-9]{3}-[0-9]{3}-[0-9]{4} # Combined pattern: (\([0-9]{3}\)\s|[0-9]{3}[.-])[0-9]{3}[.-][0-9]{4}

Part C - Date Patterns:

# ISO format: 2024-01-15 [0-9]{4}-[0-9]{2}-[0-9]{2} # More specific: [0-9]{4}-(0[1-9]|1[0-2])-(0[1-9]|[12][0-9]|3[01]) # US format: 01/15/2024 [0-9]{2}/[0-9]{2}/[0-9]{4} # More specific: (0[1-9]|1[0-2])/(0[1-9]|[12][0-9]|3[01])/[0-9]{4} # Day-Month-Year: 15-Jan-2024 [0-9]{1,2}-(Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Oct|Nov|Dec)-[0-9]{4} # More specific: (0[1-9]|[12][0-9]|3[01])-(Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Oct|Nov|Dec)-[0-9]{4}

Testing suggestions:

  • Test with regexr.com or regex101.com
  • Try edge cases like February 30th or month 13
  • Consider what happens with leading zeros

Question 6: Advanced RegEx & File Organization

Your research team has accumulated thousands of data files over several years with inconsistent naming. You need to use regex and command-line tools to reorganize and rename them systematically.

Current file examples:

Survey_2023_Jan_Raw_Data.xlsx experiment-results-2022-march-cleaned.csv data_processing_script_v3_final_FINAL.py Analysis Report March 2023.docx figure_1_experiment_A_2022.png raw-data-survey-participants-jan-2024.json Meeting_Notes_2023-04-15.txt datafile_backup_20230315.csv plot_results_v2_updated_new.pdf

Part A: Write regex patterns to identify and extract the following information from filenames:

  • Year (4 digits)
  • Month (both abbreviated like "Jan" and full like "January")
  • File type category (survey, experiment, analysis, figure, etc.)

Part B: Create a systematic renaming scheme that follows best practices. Show how you would rename each file above.

Part C: Write bash/terminal commands using regex and standard tools to:

  • Find all files containing version numbers (v1, v2, version_3, etc.)
  • List files that don't follow the YYYY-MM-DD date format
  • Group files by year and file type
Challenge Hint: Consider using tools like grep, find, ls with regex patterns, and shell loops for batch operations.

Solution:

Part A - RegEx Patterns:

# Year extraction (4 digits) [0-9]{4} # More specific - years between 2020-2029 202[0-9] # Month extraction - abbreviated (Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Oct|Nov|Dec) # Case insensitive (?i)(jan|feb|mar|apr|may|jun|jul|aug|sep|oct|nov|dec) # Month extraction - full names (January|February|March|April|May|June|July|August|September|October|November|December) # Case insensitive (?i)(january|february|march|april|may|june|july|august|september|october|november|december) # Combined month pattern (?i)(jan(uary)?|feb(ruary)?|mar(ch)?|apr(il)?|may|jun(e)?|jul(y)?|aug(ust)?|sep(tember)?|oct(ober)?|nov(ember)?|dec(ember)?) # File type category extraction ^(survey|experiment|analysis|figure|data|meeting|plot) # More flexible (?i)(survey|experiment|analysis|figure|data|meeting|plot|report)

Part B - Systematic Renaming:

# Original → Renamed Survey_2023_Jan_Raw_Data.xlsx → 2023-01-15_survey_raw-data.xlsx experiment-results-2022-march-cleaned.csv → 2022-03-15_experiment_results-cleaned.csv data_processing_script_v3_final_FINAL.py → 2023-01-15_script_data-processing_v03.py Analysis Report March 2023.docx → 2023-03-15_report_analysis-summary.docx figure_1_experiment_A_2022.png → 2022-12-01_fig01_experiment-a-results.png raw-data-survey-participants-jan-2024.json → 2024-01-15_survey_raw-data-participants.json Meeting_Notes_2023-04-15.txt → 2023-04-15_meeting_project-notes.txt datafile_backup_20230315.csv → 2023-03-15_data_backup.csv plot_results_v2_updated_new.pdf → 2023-01-15_fig02_results-updated.pdf

Naming Convention Used:

  • YYYY-MM-DD format for dates (ISO 8601)
  • Category_description format
  • Underscores for metadata separation
  • Hyphens for word separation within descriptions
  • Version numbers: v01, v02, v03 (zero-padded)
  • Figure numbers: fig01, fig02, etc.
